How it works

Three steps to full analytics coverage.

1️⃣

Add one script tag

Drop <script src="tracker.js"></script> into your HTML. The SDK self-initialises, creates a session ID in sessionStorage, and fires a page_view event immediately.

2️⃣

Tag your elements

Add data-track attributes to any buttons or links you want to track. The SDK uses event delegation — a single document-level listener catches all clicks without per-element handlers.

3️⃣

Open the dashboard

Navigate to /dashboard.html to see live stats. The dashboard polls /api/stats every 30 seconds and renders charts with pure HTML/CSS/SVG — no Chart.js, no D3.

Frequently asked questions

Does it use cookies?

No. Session IDs are stored in sessionStorage, which is cleared when the tab closes. No persistent identifiers, no consent banners needed.

How is IP anonymisation handled?

The raw IP address is never stored. Before any database write, the server computes a SHA-256 hash of the IP. The hash can detect duplicate visitors without revealing the actual address.

What happens if the network is slow?

Click and scroll events use fetch with keepalive: true. Session-end events use the Beacon API, which the browser queues and delivers even while the page is unloading.

Can I self-host this?

Yes — that's the whole point. Clone the repo, run npm install in the backend folder, and start the server. SQLite means zero external database setup.

How does rate limiting work?

The /api/events endpoint accepts a maximum of 30 requests per IP address per minute. Exceeding this returns HTTP 429. The dashboard endpoint is not rate-limited.
